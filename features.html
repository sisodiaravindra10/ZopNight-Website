Track 01

Discover.

Map your entire footprint. Every resource, every region, every account. Know what you have before you can know what to fix.

F.01 · Atlas

Multi-Cloud Atlas.

One read-only IAM role per account. Every resource scanned every 4 minutes. AWS, Azure, GCP, same screen, same filters, same provenance.

  • What it sees · Compute, storage, databases, networking, IAM, secrets, across every region the role can reach.
  • How it scans · Read-only inventory APIs · no agents, no sidecars, no code in your repo.
  • What stays out · Your data. Only metadata leaves the account. Findings + audit log live in ZopDev. Nothing else.

F.02 · Inventory diff

What changed since 4 minutes ago.

Every scan is a snapshot. Diffs are first-class. Know what was added, retired, resized, or moved between regions, with the actor who did it.

  • Replayable · Walk inventory backward in 4-minute steps for the last 90 days.
  • Attributable · Each change carries the IAM principal that ran it (when CloudTrail / Activity Log is wired).
  • Subscribable · Webhook fires on the changes you care about · per-tag, per-account, per-resource-type filters.

F.03 · Tag coverage

Coverage that names itself.

Untagged resources don't show up in showback. ZopDev surfaces accounts below tag-coverage thresholds and prompts the likely owners in Slack until they're claimed.

  • Thresholds per account · 80% default · custom per environment.
  • Inferred owners · Heuristic match against IAM creator + git blame + CODEOWNERS · suggest, never auto-assign.
  • Decay · Tags expire after 90 days unless reaffirmed · stale tags are flagged, not deleted.

F.04 · Dependency map

What touches what.

Click any resource. See every other resource that depends on it. Graph queries answer the questions invoices can't: "what blast radius if I terminate this?"

  • Three-deep · Default 3 hops; expand any node to widen.
  • Time-aware · Walk the graph at any point in the last 90 days.
  • Exportable · Cypher · GraphML · DOT · for your own analysis.

F.06 · Anomaly detection

Spend spikes, before the invoice.

Daily diff against the trailing 14-day baseline. Anomalies routed by service · grouped by likely root cause · flagged in the dashboard and Slack.

  • Per-service baselines · 3-sigma on log-scale daily spend, robust to weekend cycles.
  • Grouped causes · "All these spikes are the same Glue job, fix once" instead of 12 separate alerts.
  • Severity routing · S1 → page · S2 → Slack · S3 → digest. Per-team escalation policy.
F.07 · Forecasting

Where we end the month.

Run-rate trajectory across cost, utilization, scheduled non-prod hours. Confidence intervals visible · the band, not just the line.

  • Daily refresh · 7-day rolling LSTM with seasonal decomposition.
  • Honest bands · 80% CI shown by default · 95% on hover · no hidden assumptions.
  • What-if · Adjust schedules and remediations live; the forecast updates.

F.08 · Office-hours cron

Dev online when dev is online.

Per-environment cron in the team's timezone. Weekdays only by default · overrides per team · every schedule reversible in one click.

  • Stateful workloads · Graceful shutdown + snapshot restore. Zero data loss across the off-window.
  • Production · Never scheduled unless explicitly opted in. Default policy locks production out.

Remediate.

Findings that act on themselves. Right-size, terminate, enforce, with snapshot-before-apply rollback windows and one-click reversibility.

F.10 · One-click actions

The button that does the thing.

Every finding has an action. Not a link to a ticket. Not a Terraform PR that sits two weeks. The action runs, the resource changes, the saving books.

  • Categories · Terminate idle · right-size over-provisioned · delete orphans · enforce tags · apply policies · roll back schedules.
  • Reversibility · Snapshot-before-apply on every destructive action. 7-day rollback window. Rollback is also one click.
  • Safety rails · Dry-run by default · policy-gated · separate opt-in IAM role per action type · full audit log.

F.12 · Policy gates

Actions that only run if rules say yes.

Define policies in YAML or the visual builder. Tag-must-exist, environment-must-match, business-hours-only. No action escapes a gate.

  • Composable · Stack policies · all-of, any-of, none-of.
  • Versioned · Policies live in git when you want them to. Apply via PR.
  • Auditable · Each enforcement decision logs the policies hit.
# policies/safe-terminate.yaml
policy: safe-terminate
match:
  action: terminate-idle
requires:
  - tag-exists: owner
  - env-not-in: [prod, staging]
  - idle-for: 21d
  - approved-by: slack:@platform
on-violation: block
