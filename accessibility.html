§ 17 · wcag 2.1 aa audit

The brand, through a screen reader.

Every color combination, motion rule, keyboard path, and screen-reader label in the ZopDev system — measured against WCAG 2.1 AA. If something fails, it gets listed here with a fix, not hidden.

§ 01 · score card

Four axes. Measured.

contrast · AA 100%

All 24 text/background combinations pass at regular and large sizes.

contrast · AAA 83%

20 of 24 reach AAA. Four combos hit AA only (accent + body on muted).

motion A+

Every animation gated by prefers-reduced-motion. Loader auto-skips.

keyboard A+

Every link, button, and interactive tile reachable via Tab. Focus ring sharp.

§ 03 · motion

Motion is gated, always.

Every animation in the system is controlled by one CSS rule: @media (prefers-reduced-motion: reduce) collapses every transition and animation to 0.01ms. Declared in tokens.css at the base so it can't be forgotten per-page.

01 · loader

Tetris loader auto-skips.

If reduced-motion is on, the loader overlay is set to display:none immediately — the page renders behind it without the drop/reveal.

02 · card lift

Hover card-lift disabled.

The signature translateY(-4px) + 0 8px 0 -4px orange becomes a 1ms no-op. Border-color change still fires for focus state.

03 · section reveal

IntersectionObserver short-circuits.

Every section gets .in applied synchronously, skipping the fade/rise choreography. Page appears fully rendered.

04 · applied-toast

Toast still cycles; slide disabled.

Content rotates every 8s so users see the signal, but the slide-in/out transform becomes instant. No motion sickness.

05 · favicon pulse

Green-tile breathe stops.

.tile-green inside favicon.svg uses a CSS @media query to set animation:none when reduced-motion is on.

06 · skeleton shimmer

Loading shimmer halts.

Skeletons still show (so the reader knows content is loading), but the cream shimmer band stops moving. ::after animation becomes none.

§ 04 · keyboard

Tab order + focus ring.

Every interactive element reachable via Tab. Focus ring is a 2px orange outline (card-lift signature color, preserved even with reduced-motion). No focus-ring suppression via outline:none without a replacement.

01 · skip link

"Skip to content" on Tab.

Hidden by default, appears in the top-left on first Tab press. Jumps past the nav to the <main> element.

02 · focus ring

outline: 2px solid var(--zop-orange)

Applied on :focus-visible so keyboard users get it but mouse clicks don't flash a ring. Offset 2px from the element for readability on dark.

03 · nav

Logo first, then links, then CTA.

Natural reading order. No tabindex overrides. CTA is always the last focusable on the nav row.

04 · modals

Focus trap + ESC closes.

When a confirmation modal opens, focus moves inside and Tab cycles within. ESC closes without confirming. Return focus to trigger element on close.

05 · tetris game

All keys, no mouse required.

The 404 Tetris is keyboard-first. Arrow keys, Space, P, R all documented in the side panel. Mobile gets tap buttons.

06 · forms

Label above, error below.

Every input has a visible <label>, not just a placeholder. Errors announced via aria-live="polite".

§ 05 · screen reader

Semantic HTML first.

Heading order is strict: one h1, descending h2 → h3 → h4, never skipped. <nav>, <main>, <footer>, <section> landmarks on every page. ARIA is only added where semantics fall short — never as a decorative substitute.

01 · logo

aria-label="ZopDev home"

The SVG lockup is decorative; the link wrapping it carries the label. Screen reader says "ZopDev home, link" not "image, image".

02 · decorative SVG

aria-hidden="true"

Illustrative SVGs (hero glyphs, chart visuals, icon sprites) are hidden from assistive tech. The text around them carries the meaning.

03 · three-numbers

Labels read first.

Each number is announced as "Spent this month, $182 thousand" — mono label + figure. Never just the figure alone.

04 · live region

aria-live="polite" on the toast.

When the applied-toast cycles new content, it's announced politely (after current speech) so the user isn't interrupted.

05 · headings

One h1. Strict descent.

h1 = page title. h2 = section. h3 = subsection. h4 = card heading. Auditors get a clean outline. No h3 skipping to h5.

06 · tables

Real <table> for tabular data.

Resource lists, pricing comparisons, copy library — all use actual <thead> / <tbody>. Never flex/grid fakes.

§ 06 · touch + mobile

Minimum 44×44 px targets.

Every tappable element is at least 44×44 pixels on the first Tab focus, matching Apple HIG + WCAG 2.5.5. The .btn component at its default 13px padding + 14px text hits 44px exactly. Icon-only buttons get min-width: 44px; min-height: 44px.

01 · nav links

44px tap area, 26px visual.

Visual size stays 26px but padding extends the hit area to the full nav bar height. No accidental mis-taps.

02 · close buttons

Always 44×44 minimum.

The X-button to close a modal/toast has a 44×44 hit target even when the icon is 16px — centered inside padding.

03 · tetris mobile

Tap buttons ≥ 48px tall.

The 4-button grid on 404.html (Left / Rotate / Right / Drop) is 48px tall on mobile so gameplay isn't fiddly.

04 · form fields

Input height 44px minimum.

Every text input on mobile renders at least 44px tall, with the label stacked above (not inline).

§ 07 · known limitations

Honesty about the gaps.

Two areas where the system trades strict AAA for brand integrity. Documented here so no one is surprised.

01 · trade-off

Orange + green on cream don't hit AA body.

When the brand prints on paper (business card, report), the orange and green words are display-only (≥ 24px weight 600). Body text always falls back to ink #0A0A0A for AAA contrast.

02 · trade-off

Canonical blue #2A4494 fails AA on dark.

The brand hex is preserved for paper only. Every tile / block / large fill on dark canvas uses the lifted #4A66D4 variant. Enforced via the --logo-blue CSS variable.
